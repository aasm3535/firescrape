Documentation

FireScrape is a versatile tool that bridges the gap between the web and your applications. It functions both as a REST API for developers and a Model Context Protocol (MCP) Server for AI agents like Claude and Cursor.

Installation (MCP Server)

Install the MCP server globally to use it with your AI tools.

bun add -g @yutugyutugyutug/firescrape-mcp
npm install -g @yutugyutugyutug/firescrape-mcp
yarn global add @yutugyutugyutug/firescrape-mcp
pnpm add -g @yutugyutugyutug/firescrape-mcp

Scrape API

The scraping endpoint extracts metadata, hidden JSON structures (like Next.js data), and clean text content from any URL. It automatically handles header rotation and basic bot evasion.

GET /api/scrape
url The full URL to scrape (required). Must include http/https.

Response Structure

{
  "success": true,
  "data": {
    "url": "https://example.com",
    "meta": {
      "title": "Page Title",
      "description": "SEO Description...",
      "image": "https://...",
      "favicon": "https://..."
    },
    "structured_data": [], // Extracted JSON-LD, Next.js props, etc.
    "content_preview": "Cleaned text content..."
  }
}

What is MCP?

The Model Context Protocol (MCP) is an open standard that allows AI models (like Claude 3.5 Sonnet) to interact with external data and tools safely.

By running the FireScrape MCP server, you give your AI editor (Cursor) or assistant (Claude Desktop) the ability to:

  • Search the web for up-to-date information.
  • Read and summarize long documentation pages.
  • Browse your local files to understand your codebase context.

MCP Setup & Usage

Run Without Installing

The easiest way to start is using `bunx` or `npx`. This downloads and runs the latest version instantly.

bunx @yutugyutugyutug/firescrape-mcp

Claude Desktop Config

Add this to your `claude_desktop_config.json`:

{
  "mcpServers": {
    "firescrape": {
      "command": "bunx",
      "args": ["@yutugyutugyutug/firescrape-mcp"]
    }
  }
}

Cursor Editor Config

Go to Settings > General > MCP > Add New Server:

  • Type: Stdio
  • Command: bunx (or npx)
  • Args: @yutugyutugyutug/firescrape-mcp
